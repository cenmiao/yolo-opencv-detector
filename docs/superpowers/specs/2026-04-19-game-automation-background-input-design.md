# 游戏自动化 - 后台按键注入设计

**日期**: 2026-04-19  
**版本**: 1.0

---

## 1. 问题描述

当前使用 `pynput.keyboard` 模拟全局键盘输入，要求游戏窗口必须在前台。如果游戏窗口在后台，按键无法正确传递给游戏。

---

## 2. 解决方案

### 2.1 核心思路

在触发按键前自动检测游戏窗口状态，根据状态选择合适的输入方式：
- **前台窗口** → 使用 `pynput.keyboard`（全局输入）
- **后台窗口（可见）** → 使用 `SendMessage`（后台注入）
- **窗口最小化** → 尝试恢复；失败则提示用户并暂停程序

### 2.2 技术选型

| 组件 | 说明 |
|------|------|
| 窗口状态检测 | `win32gui.GetForegroundWindow()` 判断是否前台 |
| 窗口最小化检测 | `win32gui.IsIconic()` |
| 窗口恢复 | `win32gui.ShowWindow(hwnd, SW_RESTORE)` |
| 后台按键注入 | `win32api.SendMessage(hwnd, WM_KEYDOWN/WM_KEYUP, vk, 0)` |
| 虚拟键码映射 | 常用键硬编码 + `win32api.VkKeyScan()` 回退 |

## 3. 架构设计

### 3.1 组件改动

#### AutomationEngine
- `__init__`：新增 `hwnd` 参数
- `trigger_action`：增加窗口状态判断逻辑，分派到不同触发方法
- 新增 `_send_message_key`：封装 SendMessage 按键注入

#### WindowCapture
- 无需改动，`self.hwnd` 已存在

## 4. 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 窗口句柄无效 | 记录日志，跳过本次触发 |
| SendMessage 失败 | 记录日志，尝试回退到 pynput |
| 窗口恢复失败 | 显示提示，等待用户按回车继续或退出 |
| VkKeyScan 查询失败 | 使用默认 VK 码 0x41 (A) 并警告 |

---

## 5. 测试要点

1. **前台窗口测试**：游戏在前台，验证 pynput 正常工作
2. **后台窗口测试**：游戏在后台，验证 SendMessage 有效
3. **最小化恢复测试**：窗口最小化时验证能否自动恢复
4. **按键映射测试**：验证常用键（space, enter, x, z 等）正确触发
