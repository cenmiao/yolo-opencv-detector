# 游戏自动化检测工具设计文档

**日期**: 2026-04-19  
**版本**: 1.0

---

## 1. 概述

基于已有的 YOLO OpenCV 检测器代码，开发一个游戏自动化辅助工具。该工具使用训练好的 YOLOv4-tiny 模型检测游戏中的三类目标：
- `1ziji` - 自己的角色
- `2diguihanghui` - 敌对行会的玩家
- `3zijihanghui` - 自己行会的玩家

当检测到敌对行会玩家进入指定距离阈值时，自动触发键盘按键（如技能释放）。

---

## 2. 功能需求

### 2.1 窗口选择
- 启动后列出所有可用的 Windows 窗口
- 用户通过编号选择目标窗口
- 支持刷新窗口列表和重新选择

### 2.2 配置持久化
- 配置文件名：`runtime-config.json`
- 保存内容：
  - 窗口名称
  - 配置文件路径（.cfg）
  - 权重文件路径（.weights）
  - 检测频率（FPS）
  - 距离阈值（像素）
  - 触发按键
- 下次启动时自动加载上次的配置

### 2.3 检测频率
- 用户可手动设置检测频率（FPS 或间隔毫秒）
- 每次截图后执行一次检测
- 频率与截图同步

### 2.4 自动化逻辑
- 实时检测自己角色与敌对行会玩家的距离（2D 像素距离）
- 当距离 < 阈值且冷却时间结束时触发按键
- 冷却时间：500 毫秒
- 多目标处理：优先锁定距离最近的敌人

### 2.5 可视化选项
- 用户可选择是否开启可视化窗口
- 可视化内容与检测同步刷新
- 运行中可按 'v' 键切换可视化开启/关闭
- 绘制内容参考 `4_yolo_opencv_detector.ipynb`：
  - 所有检测目标的边界框
  - 自己角色与最近敌人的距离线
  - 距离数值显示
  - 阈值范围提示
  - 当前状态（冷却中/可触发）

### 2.6 打包发布
- 打包脚本：`build_automate.bat`
- 输出文件：`dist/yolo_automate.exe`
- 外部文件（不打包进 EXE）：
  - `yolov4-tiny-custom.cfg`
  - `yolov4-tiny-custom_last.weights`
  - `yolov4-tiny/obj.names`

```
│  │ 5. 自动化参数（距离阈值、按键）→ 保存配置              │  │
│  │ 6. 可视化选项                                         │  │
│  └──────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌────────────────────────────────────────────────────────────┐
│                    检测循环核心                              │
│  while running:                                            │
│    1. get_screenshot()                                     │
│    2. process_image() → YOLO 推理                           │
│    3. 过滤三类目标：自己、敌对、友方                        │
│    4. 计算自己与最近敌人的距离                              │
│    5. if 距离 < 阈值 and 冷却结束 → 触发按键               │

│    7. 按频率等待                                           │

```

### 3.2 组件设计

#### WindowCapture（已有，需增强）
- `list_available_windows()` - 列出所有可见窗口
- `__init__(window_name)` - 初始化窗口捕获
- `get_screenshot()` - 获取窗口截图
- `get_window_size()` - 获取窗口尺寸

#### ImageProcessor（已有）
- `__init__(img_size, cfg_file, weights_file)` - 加载 YOLO 模型
- `process_image(img)` - 处理图像并返回检测结果
- `get_coordinates(outputs, conf)` - 解析模型输出
- `draw_identified_objects(img, coordinates)` - 绘制检测结果

#### ConfigManager（新增）
- `load_config()` - 从 runtime-config.json 加载配置
- `save_config(config)` - 保存配置到文件
- `get_default_config()` - 获取默认配置

#### AutomationEngine（新增）
- `__init__(distance_threshold, key_to_press, cooldown_ms)` - 初始化
- `calculate_distance(center1, center2)` - 计算两点像素距离
- `find_nearest_enemy(player, enemies)` - 找到最近的敌人
- `should_trigger(distance, current_time)` - 判断是否应该触发按键
- `trigger_action(keyboard)` - 执行按键动作

## 6. 错误处理

### 6.1 窗口相关
- 窗口不存在：提示用户重新选择
- 窗口已最小化：提示用户恢复窗口
- 窗口尺寸无效：提示用户检查窗口状态

### 6.2 文件相关
- 配置文件不存在：使用默认值并提示
- 权重文件不存在：报错退出
- 配置文件读取失败：使用默认值并提示

### 6.3 运行时
- 模型加载失败：报错退出并显示详细错误
- 截图失败：尝试恢复，连续失败则退出
- 按键触发失败：记录日志，继续运行
