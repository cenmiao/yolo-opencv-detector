# 游戏自动化工具设计文档

## 项目概述

将 YOLO 检测模型与 pynput 结合，实现游戏自动化辅助工具。支持两种自动化模式，打包成 exe 独立运行。

## 目标检测类别

| 类别 ID | 类别名称 | 说明 |
|---------|----------|------|
| 0 | ziji | 自己的角色 |
| 1 | dangeroususer | 危险玩家（敌人） |
| 2 | normaluser | 普通玩家（忽略） |

## 核心需求

1. 打包成 exe 运行
2. 启动时列出所有可用窗口，用户选择目标窗口
3. 配置信息保存到 `auto-run.json`，下次启动自动加载
4. 支持两种自动化模式，启动时选择
5. 图像绘制显示自己角色的坐标位置

## 配置文件结构

文件名：`auto-run.json`

```json
{
  "window_name": "游戏窗口标题",
  "cfg_file": "./yolov4-tiny/yolov4-tiny-custom.cfg",
  "weights_file": "./yolov4-tiny/yolov4-tiny-custom_last.weights",
  "mode": 1,
  "distance_threshold": 200,
  "trigger_key": "space",
  "fixed_position": {
    "x": 400,
    "y": 300
  }
}
```

### 配置项说明

| 字段 | 类型 | 说明 |
|------|------|------|
| window_name | string | 目标窗口标题 |
| cfg_file | string | YOLO 配置文件路径 |
| weights_file | string | YOLO 权重文件路径 |
| mode | int | 自动化模式（1 或 2） |
| distance_threshold | int | 触发距离阈值（像素） |
| trigger_key | string | 触发按键名称 |
| fixed_position | object | 模式二使用的固定坐标 |

## 自动化模式

### 模式一：基于自己角色检测

**检测逻辑**：
1. 从检测结果中找到 `ziji`（自己角色）
2. 从检测结果中找到所有 `dangeroususer`（危险玩家）
3. 计算自己角色与每个危险玩家的距离
4. 取距离最近的危险玩家
5. 如果距离 < 阈值 → 触发按键

**特殊情况**：
- 未检测到自己角色：不触发，继续检测
- 未检测到危险玩家：不触发，继续检测

### 模式二：基于固定坐标

**用途**：当模型无法准确识别自己角色时，使用手动配置的固定位置替代。

**检测逻辑**：
1. 使用配置的 `fixed_position` 作为自己角色位置
2. 从检测结果中找到所有 `dangeroususer`
3. 计算固定坐标与每个危险玩家的距离
4. 取距离最近的危险玩家
5. 如果距离 < 阈值 → 触发按键

**固定坐标获取方式**：
- 通过模式一运行时打印的坐标数据来确定
- 用户手动测量后输入

## 按键触发

**触发方式**：单次按键
- 每次距离小于阈值时发送一次按键
- 无冷却时间限制

**按键发送机制**：
- 窗口前台：使用 pynput 发送
- 窗口后台：使用 win32api.SendMessage 发送

## 可视化绘制

在检测窗口显示：

1. 所有检测对象的边界框（不同类别不同颜色）
2. 自己角色的中心坐标文本显示
3. 最近危险玩家与自己角色的连线
4. 当前距离数值
5. 距离阈值圆（以自己角色中心为圆心）
6. 当前使用的模式（1 或 2）
7. FPS 信息

### 颜色方案

| 类别 | 颜色 (BGR) |
|------|------------|
| ziji | (0, 255, 0) 绿色 |
| dangeroususer | (0, 0, 255) 红色 |
| normaluser | (255, 255, 0) 黄色 |

## 启动流程

```
┌─────────────────────────────────────┐
│ 1. 加载 auto-run.json 配置           │
│    (不存在则使用默认值)               │
├─────────────────────────────────────┤
│ 2. 显示可用窗口列表                   │
│    用户选择目标窗口                   │
├─────────────────────────────────────┤
│ 3. 配置文件路径选择                   │
│    - cfg 文件路径                    │
│    - weights 文件路径                │
│    (已有配置可直接回车使用)           │
├─────────────────────────────────────┤
│ 4. 选择自动化模式                     │
│    - 输入 1: 模式一                  │
│    - 输入 2: 模式二                  │
├─────────────────────────────────────┤
│ 5. 设置距离阈值                       │
│    (已有配置可直接回车使用)           │
├─────────────────────────────────────┤
│ 6. 设置触发按键                       │
│    (已有配置可直接回车使用)           │
├─────────────────────────────────────┤
│ 7. 如果模式二: 设置固定坐标           │
│    - X 坐标                          │
│    - Y 坐标                          │
├─────────────────────────────────────┤
│ 8. 保存配置到 auto-run.json          │
├─────────────────────────────────────┤
│ 9. 初始化 YOLO 模型                  │
├─────────────────────────────────────┤
│ 10. 开始自动化检测循环               │
│     按 'q' 退出                      │
└─────────────────────────────────────┘
```

## 运行时交互

| 按键 | 功能 |
|------|------|
| q | 退出程序 |
| v | 切换可视化窗口显示 |

## 打包要求

使用 PyInstaller 打包：
- 单文件模式或目录模式
- 包含 yolov4-tiny 目录下的配置文件
- 包含 obj.names 类别文件

## 代码架构

```
game_automate.py
├── ConfigManager        # 配置管理
├── WindowSelector       # 窗口选择
├── WindowCapture        # 窗口截图
├── ImageProcessor       # YOLO 检测
├── AutomationEngine     # 自动化引擎
│   ├── calculate_distance()
│   ├── find_nearest_enemy()
│   ├── trigger_action()
│   └── mode1_process()  # 模式一处理
│   └── mode2_process()  # 模式二处理
└── main()               # 主入口
```

## 错误处理

1. 窗口未找到：提示用户重新选择
2. 窗口最小化：暂停检测，等待窗口恢复
3. 模型加载失败：提示检查文件路径
4. 截图失败：记录日志，继续尝试
